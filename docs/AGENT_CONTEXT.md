# SquarePic — Project Context for AI Agents

This document provides a comprehensive overview of the **SquarePic** project to ensure any AI assistant can quickly understand the codebase, architectural decisions, and deployment workflows.

## 1. Project Overview
*   **Purpose:** A privacy-first, high-performance online tool to convert any image into a perfect 1:1 square format without cropping important content.
*   **Core Philosophy:** "Local Processing." All image editing happens in the user's browser via HTML5 Canvas. Images are never uploaded to a server or stored in the cloud.
*   **Target Audience:** Content creators, social media managers (Instagram/LinkedIn), and ecommerce sellers.

## 2. Technical Stack
*   **Frontend:** Vite + TypeScript.
*   **Rendering:** HTML5 Canvas API for image manipulation (blur backgrounds, solid fills, scaling).
*   **Styling:** Vanilla CSS with a focus on modern aesthetics (Cyan-default theme, glassmorphism, dark mode).
*   **Deployment:** Hostinger (Apache) via GitHub Actions (FTP Sync).

## 3. Directory Structure & Repository Strategy
The repository is uniquely structured to facilitate professional local development while maintaining a "production-clean" Git history for deployment.

*   **`/src`**: Contains TypeScript source files and the main CSS design system.
*   **`/public_html`**: The **Source of Truth** for production. This folder contains the compiled assets (`/assets`), static pages (`about.html`, `faq.html`, etc.), and the `.htaccess` file.
*   **`.gitignore`**: Highly customized. It is configured to track primarily the `public_html/` and `.github/` directories. This ensures that the repository remains lightweight and deployment-ready at all times.
*   **`.github/workflows/deploy.yml`**: Automates the sync of the `public_html/` folder to the Hostinger server root via FTP.

## 4. Key Implementation Details
*   **Theme System:** Uses CSS root variables (`--accent`, `--accent-glow`). Default is **Cyan** (`#06b6d4`). Themes are persisted in `localStorage`.
*   **Security:** A robust `.htaccess` file enforces HTTPS, HSTS, and strict Content Security Policy (CSP) headers to prevent XSS and clickjacking.
*   **SEO:** Every page is optimized with high-converting copy, descriptive meta tags, and structured headings targeting keywords like "free online square image maker."

## 5. Deployment Workflow
To update the live site at `squarepic.io`:
1.  Run `npm run build` locally.
2.  Sync the `dist/` output to the `public_html/` folder (on Windows: `robocopy dist public_html /MIR`).
3.  Commit and push the changes in `public_html/` to the `main` branch on GitHub.
4.  The GitHub Action triggers automatically to sync the folder to Hostinger.

## 6. Current State & Known Constraints
*   **RemoveBG:** The tool uses `@imgly/background-removal` for client-side AI background removal. This is hardware-intensive and may be slower on older devices.
*   **Git State:** The project is currently in a "Production Ready" state with finalized SEO copy and branding.
